# natori-japanese

voice-ai-partnerのキャラクター「名取」の返答が、キャラクター仕様に沿っているかを評価するスキル。[coji/natural-japanese](https://github.com/coji/natural-japanese)(ビジネス文書向けの日本語自然さ評価スキル)の設計思想 — 「検出は機械、判断はAI」「事後修正より生成時制約」 — を踏襲しつつ、対象を「長文のビジネス文書」から「音声会話の短い1ターン返答」に絞って再設計した、独立した新しいスキル。natural-japanese本体のファイルは一切変更していない。

## natural-japaneseとの違い

natural-japaneseは議事録・レポート・ガイドといった長文文書を対象に、12条の文体憲法・doctype別テンプレート・構造レビュー・sudachipyによる形態素解析ベースの厳密なlintを備える。本スキルはそれとは異なり:

- 対象は名取という**特定のキャラクター**の**短い音声返答**(実測40〜100文字程度)のみ
- 文体憲法やdoctypeは不要 — 唯一の根拠は `references/character-natori.md`(本番のシステムプロンプトのコピー)
- lintはsudachipy等の形態素解析に依存せず、**標準ライブラリのみ**の正規表現・文字列マッチで実装(`scripts/lint_natori.py`)。短い会話文であれば、これで十分な精度が出る
- 評価項目はユーザー指定の10項目に固定(`references/evaluation-criteria.md`)

## 使い方

- `/natori-japanese score <ファイルまたはテキスト>` — 診断のみ。返答(または会話ログ)を書き換えず、10項目のスコアと理由を返す。**最初に必ず `references/character-natori.md` と `references/evaluation-criteria.md` を読む**
- `/natori-japanese write <お題>` — 名取らしい返答のサンプルを新規に書く(プロンプト調整の試作・比較用。本番の返答生成はバックエンドのLambdaが行うため、これは開発時の試作にのみ使う)

引数なしで自然文で呼ばれた場合も同様に判断する(「名取っぽいか採点して」→ score 相当、「こういう返答を書いて」→ write 相当)。

## 対象の形式

- **単発の返答**(1行のテキストまたは1行だけのファイル): 語尾・一人称・呼称・敬語・AI臭・長さ・キャラクター崩壊を評価する。「7. 同じ文型の繰り返し検出」は複数ターンがないと意味を持たないため、単発評価では対象外とする
- **複数ターンの会話ログ**(1行1ターンのテキストファイル、例えば実機テストの録音を書き起こしたもの): 上記に加えて、ターンをまたいだ語尾・文型の反復も検出できる

## score モードの手順

1. `references/character-natori.md` を読み、キャラクター仕様を把握する(初回、または前回の読み込みから内容が変わっていそうなら必ず読み直す)
2. `references/evaluation-criteria.md` を読み、10項目それぞれの基準と判定の閾値を把握する
3. 機械的検出を実行する:
   ```
   python3 scripts/lint_natori.py <file> --json
   ```
   `uv` 等の追加ツールは不要(標準ライブラリのみ)。ファイルが手元にない場合は対象テキストを一時ファイルに書き出してから渡す
4. lintのfindingsは疑いの提示であり、これをそのまま転記して終わりにしない。特に「参考」レベルのfindings(テンポのフィラー未検出など)は文脈で判断する。lintが拾わない項目(2. テンポの良し悪しそのもの、9. の間接的なキャラクター崩壊)は、`character-natori.md` を下敷きに目視で評価する
5. `references/evaluation-criteria.md` 末尾の出力フォーマットに従って、10項目の判定表・総合スコア・致命的な問題・(要求があれば)修正提案をまとめる
6. 書き換えは頼まれるまで行わない。診断のみで完了する

## write モードの手順

1. `references/character-natori.md` を読み、指定されたお題に対する名取らしい返答を書く。音声会話ルール(一度に完璧な回答を返さない、質問されていない内容まで説明しない)を厳守する
2. 書いたら自分で `scripts/lint_natori.py` にかけ、findingsが出なくなるまで(または致命的なfindingsが残っていないことを確認した上で)微調整する
3. 最終的な返答テキストと、簡単な自己評価(致命的な問題がないことの確認)を返す

## 本番プロンプトとの同期

`references/character-natori.md` は `infra/lambda/conversation/system-prompt.md` の手動コピーである。本番のキャラクター仕様を編集したら、このファイルも同じ内容に更新すること(自動同期の仕組みはない)。同期が取れていないと、このスキルの評価基準が本番の挙動とズレる。

## 後片付け

score/writeの過程で作った一時ファイル(会話ログの書き出し等)は、作業完了後に削除する。
